Referral Conversions & Commission Tracking

Every time someone registers or joins using an affiliate's code, a conversion row is created automatically — same trigger pattern already used for promo_codes.used_count.

referral_conversions Links affiliate → promo_code → the resulting registration or membership, with sale amount, calculated commission, and payout status.

How a Conversion Is Created

1. Trigger
Buyer uses code
A registration or membership payment uses an affiliate's promo code at checkout.
2. Auto-detect
DB trigger fires
Same pattern as today's promo_codes.used_count trigger — checks if the code has an affiliate_id.
3. Record
Conversion row created
sale_amount × commission_rate = commission_amount, status = pending.
4. Payout
Admin marks Paid
Same manual-verify pattern as event payments today — admin reviews, then pays out.
